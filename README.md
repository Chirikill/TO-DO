# TO-DO
to-do website(application) to suit mine needs
## Введение
**Тип проекта:** Веб-приложение (Single Page Application)
**Цель:** Помочь пользователям запоминать информацию с помощью метода интервального повторения. Пользователь добавляет темы, система автоматически рассчитывает даты повторений (через 1 день, 3 дня, 7 дней, 30 дней) и отображает их в календаре.

**Время выполнение:** ~8 ч.
## 1. Настройка окружения: Git в WSL, SSH-аутентификация, VS Code

   Используемые инструменты:
   - WSL (Windows Subsystem for Linux) — Ubuntu 20.04/22.04
   - Git — версия 2.43.0 (установлен внутри WSL)
   - VS Code — с расширением Git (Source Control)
   - GitHub — удалённый хостинг репозитория


### 1.1 Установка и настройка Git в WSL
Для устранения проблем с компонентом `remote-https` Git был полностью удалён и переустановлен:
#### 1. Удаление Git и конфигураций
```bash
sudo apt remove --purge git
```
Удаляет пакет Git и все его системные конфигурационные файлы.

#### 2. Очистка неиспользуемых зависимостей
```bash
sudo apt autoremove
```
Удаляет пакеты, которые были установлены как зависимости для Git, но больше не нужны.

#### 3. Очистка кэша пакетов
```bash
sudo apt clean
```
Очищает локальный кэш загруженных пакетов, освобождая место на диске.

#### 4. Обновление списка пакетов (опционально)
```bash
sudo apt update
```
Обновляет список доступных пакетов из репозиториев.

### 1.2 Настройка SSH-аутентификации
При попытке выполнить git push через VS Code возникала ошибка:

#### Текст ошибки
```
git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.
```

#### Причина
SSH-ключ не добавлен в GitHub или отсутствует на локальной машине.
#### Решение

1. Проверить наличие SSH-ключа:
   ```bash
   ls -la ~/.ssh/
   ```

2. Если ключа нет, создайть новый:
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```

3. Добавить ключ в SSH-агент:
   ```bash
   eval "$(ssh-agent -s)"
   ssh-add ~/.ssh/id_ed25519
   ```

4. Скопировать публичный ключ:
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```

5. Добавьте ключ в GitHub: Settings → SSH and GPG keys → New SSH Key
   
**Результат:** VS Code получил доступ к SSH-ключу, и команды git push / git pull заработали без запроса пароля.

### **Функциональные возможности на 2 этапе:**
#### Добавление темы
- Поле ввода и кнопка "Добавить"
- Добавление по клавише Enter
- Каждая тема получает уникальный ID (на основе временной метки)

#### Система интервального повторения
- При добавлении темы фиксируется дата изучения
- Автоматически рассчитываются даты повторений:
  - Сегодня (день добавления):
  - Через 1 день
  - Через 3 дня
  - Через 7 дней
  - Через 30 дней
  * при каждом повторении интервал обнуляется

#### Календарь
- Отображение текущего месяца
- Навигация по месяцам (кнопки)
- Клик по дню → отображение списка задач на этот день
  
#### Список "Повторить сегодня"
- Автоматически отображает темы, запланированные на сегодня
- Кнопка "Выучено!" для отметки темы как изученной

#### Список "Все темы"
- Отображает все добавленные темы
- Статус-бейджи:  Повторить,  Ожидает,  Изучено
- Кнопка удаления (активна только для невыученных тем)

#### Хранение данных
- Все данные сохраняются в localStorage
- Данные не теряются при перезагрузке страницы

## 4.  CI/CD: Автоматический деплой на GitHub Pages
Цель: автоматизировать процесс публикации сайта при каждом пуше в ветку main

  Используемые инструменты:
  - GitHub Actions
  - GitHub Pages
  - YAML
  - Artifacts

### 4.1 Что происходит при каждом пуше
1. Триггер — git push в ветку main автоматически запускает workflow.
2. Временный сервер — GitHub создаёт чистый сервер с Ubuntu.
3. Клонирование — сервер скачивает код из репозитория (actions/checkout).
4. Сборка артефакта — все файлы сайта упаковываются в архив (upload-pages-artifact).
5. Деплой — архив передаётся в GitHub Pages (deploy-pages).
6. Публикация — сайт становится доступен по адресу: https://chirikill.github.io/TO-DO/

### 4.2 Решение проблем при настройке

#### Ошибка 403 (Permission denied)

**Причина:** Недостаточно прав у GitHub Actions для записи в репозиторий.
**Решение:** В настройках репозитория было включено `Read and write permissions` для GitHub Actions.
**Путь:** `Settings` → `Actions` → `General` → `Workflow permissions`


#### Ошибка Missing environment

**Причина:** В workflow-файле не указано окружение для деплоя.
**Решение:** Был добавлен блок `environment: name: github-pages` в файл `deploy.yml`.

#### Токен без права workflow

**Причина:** Используемый Personal Access Token не имеет разрешения на управление workflow.
**Решение:** Был создан новый Personal Access Token с правами `repo` и `workflow`.

#### Конфликт веток при пуше

**Причина:** Локальная и удалённая ветки имеют разную историю коммитов.
**Решение:** Был использован `git push --force` для перезаписи истории.
